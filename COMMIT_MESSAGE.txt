feat: SHIPS v0.2.0 — system scope, configurable validation, programmatic tokenisation

Introduces system/environment deployment scope distinction, five new
object types (MAP, AUTHORIZATION, FOREIGN_SERVER, JAR, STO), the
00_system deployment phase with SKIP_IF_EXISTS semantics, configurable
validation rules via inspect.conf, environment-aware filename resolution
at package time, and programmatic tokenisation via --generate-token-map
and --token-map on the harvest command.

New features:
- DeployScope.SYSTEM / ENVIRONMENT classification
- SKIP_IF_EXISTS deploy strategy for system-scope objects
- 00_system phase (Maps, Roles, Profiles, Authorisations, Foreign Servers)
- Configurable inspect rules (inspect.conf: ERROR / WARNING / OFF)
- Builder resolves filenames from DDL content at package time
- Token map generation (--generate-token-map, --env-prefix, --token-map)
- C source co-artefact support (.c/.h alongside function DDL)
- System existence queries (DBC.RoleInfoV, MapsV, ProfileInfoV, etc.)
- 368 unit tests across 9 modules

Breaking changes:
- ROLE/PROFILE strategy changed from DIRECT_EXECUTE to SKIP_IF_EXISTS
- File extensions changed: ROLE→.rol, PROFILE→.prf, USER→.usr, GRANT/REVOKE→.dcl
- validate_directory() signature changed: rules_config dict replaces boolean flags
- Scaffolder directory structure: system objects moved from DDL/ to system/
